# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build and test

Scala 3.3.7, sbt 1.12.11, JDK 11 (Temurin). Tests use MUnit + munit-scalacheck.

- `sbt compile` / `sbt test` — standard build and test.
- `sbt ci` — custom task defined in `build.sbt` that runs the exact sequence GitHub Actions runs (`scalafmtSbtCheck`, `scalafmtCheckAll`, then `tpolecatCiMode compile`, then `tpolecatCiMode test`). Use this before pushing to catch what CI will catch.
- `sbt "testOnly cronh.domain.fieldTypes.MinuteTest"` — run a single test class. Append `-- --tests=*pattern*` to filter test names.
- `sbt scalafmtAll` — format all sources (CI fails on unformatted code).

`tpolecatCiMode` upgrades warnings to errors, so a clean `sbt compile` locally may still fail in `ci` mode. Run `sbt ci` to verify.

The GitHub Actions workflow is generated by `sbt-github-actions` — do not edit `.github/workflows/ci.yml` by hand; change `githubWorkflowBuild` in `build.sbt` and regenerate.

## Architecture

A Scala 3 cron library with three layers: a strictly-typed domain ADT, a dialect-aware renderer,
and a phase-ordered fluent DSL on top. There is no cron-string *parser* yet (round-tripping is a
future goal). Source is organized into three packages.

### `cronh.domain` — the ADT

1. **`Term[+A]`** (`Term.scala`) — one cron token: `All` (`*`), `Single(value)`, or
   `Range(from, to)`. `Range.apply` is a smart constructor requiring `from <= to` via
   `Ordering[A]`; the case-class constructor is private so the check can't be bypassed. `Term` is
   covariant so `Term.All: Term[Nothing]` assigns anywhere. There is deliberately no `OneOf` term —
   composition lives on `Field` (DESIGN §2.4).

2. **`Field[+A]`** (`Field.scala`) — a non-empty list of `Term[A]`, one comma-separated cron field
   (`1,5-10,15`). The non-empty invariant is enforced by a private constructor over `::[Term[A]]`
   (the cons-cell type, not `List`) plus factory methods: `Field.all`, `Field.single`,
   `Field.range`, `Field.of` (multiple singles), `Field.from` (mixed term shapes). `++`
   concatenates and stays non-empty by construction.

3. **`CronExpression`** (`CronExpression.scala`) — a plain five-field record typed by unit:
   `Field[Minute]`, `Field[Hour]`, `Field[DayOfMonth]`, `Field[Month]`, `Field[DayOfWeek]`. No
   phantom type parameters — the earlier phantom-tag design was removed (DESIGN §2.9).

4. **Normalization** (the `normalized` / `isNormalized` extensions in `Field.scala`) — opt-in,
   requiring `Ordering[A]` and `DomainBounds[A]`. Redundant/overlapping/adjacent terms
   (`*,1`, `1-5,3-7`, `1-3,4-6`) are **accepted as valid input** and canonicalized only on demand,
   never at construction (DESIGN §2.7, §4.4).

### `cronh.domain.fieldTypes` — the time-unit types

The *values* that appear in a cron field, each its own validated type:

- **`Minute` (0–59), `Hour` (0–23), `DayOfMonth` (1–31)** — private-constructor case classes
  wrapping a validated `Int` (`final case class X private (value: Int) derives CanEqual`), with a
  smart-constructor `apply` that `require`s the bounds and throws `IllegalArgumentException` naming
  the field and bad value. Bounds live as `inline val MinValue`/`MaxValue` on the companion so the
  compile-time literal macros share a single source of truth (DESIGN §2.2). Nominal case-class
  identity is what keeps cross-unit comparisons false (`Minute(5) == Hour(5)` is `false`) — the
  types were once opaque aliases over `Int` and shared `Int`'s universal equality, which answered
  `true`.
- **`Month`** — a Scala 3 enum carrying `value: Int` (January = 1) for ordering.
- **`DayOfWeek`** — a Scala 3 enum (Monday-first) with **no numeric encoding**: day numbering
  (0–6 vs 1–7, Sun-first vs Mon-first) is dialect-specific and belongs in the renderer, not the
  type (DESIGN §2.15).
- Each companion provides a `given Ordering` (required by `Term.Range` and `Field.range`) and a
  `given DomainBounds` (the full ascending domain, used by normalization). `MonthDay` was renamed
  `DayOfMonth` to match the `dayOfMonth`/`dayOfWeek` field names and the `.onThe`/`.orThe` DSL verbs.

### `cronh.render` — dialect-aware rendering

- **`Render[A]`** (`Render.scala`) — per-unit token typeclass, plus `renderTerm`/`renderField`.
- **`CronDialect`** (`CronDialect.scala`) — owns the dialect-specific `Render[DayOfWeek]` (Sun=0
  vs Sun=1 lives here, not on the enum) and a `render(CronExpression): String`. An overridable
  `renderDayOfWeek` hook lets the Unix dialect split Sunday-terminated weekday ranges (`5-6,0`
  instead of the invalid `5-0`, DESIGN §2.20).
- **`UnixCronDialect`** (`UnixCronDialect.scala`) — the default. `CronDialect`'s companion
  provides `given default: CronDialect = UnixCronDialect`, so `expr.toCron` works with no import;
  a local given shadows it to select another dialect.
- Extensions on `CronExpression`: `toCron(using CronDialect): String` and
  `humanReadable: String` (`HumanReadable.scala`, which recognizes weekday/weekend idioms and
  phrases the day-OR case as "X or Y").

### `cronh.dsl` — the phase-ordered fluent DSL

The read-aloud surface (`Schedule.scala`). It builds a schedule in one fixed order —
**month → day → time** — enforced by intermediate phase types, so an out-of-order call is a
missing-method compile error (DESIGN §2.9):

- `Schedule` (entry) → `.in(Month, …)` : `MonthChosen`
- day phase: `.daily` : `TimePhase`; `.weekdays`/`.weekends`/`.on(DayOfWeek, …)` : `DayOfWeekChosen`;
  `.onThe(DayOfMonth, …)` : `DayOfMonthChosen`
- the day-field OR is reachable only by name: `DayOfWeekChosen.orThe(…)` and
  `DayOfMonthChosen.orOn(…)` → `TimePhase` (DESIGN §2.12). `.on` and `.onThe` cannot be combined
  as independent calls.
- time phase (terminal): `TimePhase.at(h: Hour, m: Minute)` / `at(t: Time)` → `CronExpression`.

Internally the DSL accumulates into `CronExpressionBuilder`, whose five fields are `Option[Field[_]]`
(`None` = "user said nothing about this position"); only the terminal `build()` resolves `None` to
`Field.all`, keeping the implicit-`*` default visible in one place (DESIGN §2.10).

Supporting pieces in the same package:

- **`Time`** (`Time.scala`) — `Time(hour, minute)` canonical 24h wall-clock value; `Time.parse`
  accepts 12h/24h strings and returns `Either[TimeParseError, Time]`. A `time"…"` interpolator
  exists but validates at **runtime** for now.
- **`literals.scala`** — compile-time-validated literal macros `9.h`, `30.min`, `15.dom`; an
  out-of-range literal is a compile error, bounds sourced from the `inline val`s above.
- **`TimeMacro.scala`** — a compile-time validating macro for the `time"…"` interpolator, written
  but **not yet wired** (DESIGN §6 Q4).
- **`aliases.scala`** — `Mon`..`Sun` / `Monday`..`Sunday`, `midnight`, `noon`.
- **`WeekdaySelector.scala`** — wildcard-free day selection (`Mon to Fri`, `Weekdays`, `Weekends`,
  `++`); exists but is **not yet wired** to `.on` (DESIGN §6 Q3).

**In-flux DSL surface** (implemented but of undecided status — see DESIGN §5 / §6): the
vararg-hours `at(h, hs*)` overload (silently pins minute 0), `everyMinute`, and the unimplemented
`between(lo, hi)` (`???`). Don't treat these as settled API.

### Conventions to preserve

- **Smart constructors over public constructors.** New units or term shapes follow the existing
  pattern: private case-class constructor, validation in `apply`, `IllegalArgumentException`
  naming the field and bad value. Never relax a private constructor to make a test easier — fix
  the test.
- **No dialect logic in the domain.** Anything that varies by cron flavor (POSIX vs Quartz vs
  Vixie, day-of-week numbering, `?`, `L`, `W`, `#`, step syntax) belongs in `cronh.render` (or a
  future parser), not in `cronh.domain`. The numeric-less `DayOfWeek` and the Sunday-range split
  in `UnixCronDialect` are the canonical examples.
- **Redundancy is accepted, not rejected.** Only real invariants (backwards ranges, domain
  violations, empty fields) are rejected at construction. Overlaps and duplicates are valid input
  surfaced through opt-in `normalized`. Consult `DESIGN.md` §4 before introducing validation or
  normalization that would resolve an open edge-case question implicitly.
